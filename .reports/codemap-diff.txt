BenchmarkProgress Widget:
  • State machine: BenchmarkState(Enum) with values IDLE, RUNNING, DONE, ERROR, CANCELLED
  • Render:
    - Horizontal header: Label (state indicator) + Button (Cancel, disabled when IDLE)
    - RichLog: real-time output (highlight=True, markup=True, wrap=True)
  • API:
    - run_command(cmd: list[str], cwd: str | None) → starts subprocess
    - cancel() → public cancel method
  • Implementation:
    - _stream_subprocess(cmd, cwd) → @work(thread=True) decorator (runs on thread worker)
    - Uses subprocess.Popen(cmd, start_new_session=True) to create process group
    - On cancel: os.killpg(os.getpgid(pid), signal.SIGTERM) or fallback terminate()
    - Thread-safe updates: app.call_from_thread(query_one, "#bp-log", RichLog).write(line)
    - Exit code check: 0 → Done, non-zero → Error, or state==CANCELLED → Cancelled

ResultTable Widget:
  • Simple wrapper around DataTable
  • Init: title: str = "" (optional label)
  • Compose: Label (if title) + DataTable(id="dt")
  • API: load(columns: list[str], rows: list[list]) → clears + reloads data
  • DataTable.add_columns(*columns), then for each row: add_row(*[str(v) for v in row])

DATA FLOW PATTERNS
================================================================================

Benchmark Execution:
  BenchmarksPanel (user selects options)
    ↓
  Button.Pressed event → run_vdb() (or run_rag, run_emb, run_llm)
    ↓
  Build command: uv run python benchmarks/*/evaluate.py --args
    ↓
  BenchmarkProgress.run_command(cmd, cwd)
    ↓
  _stream_subprocess() @work(thread=True)
    ├─ subprocess.Popen() with start_new_session=True
    ├─ Read stdout line-by-line
    └─ app.call_from_thread() → RichLog.write() (main thread)
    ↓
  Benchmark script writes result JSON to benchmarks/*/results/TIMESTAMP.json
    ↓
  User clicks Results tab
    ↓
  ResultsPanel.on_show() → _load()
    ├─ _latest_json(directory) finds newest *.json
    ├─ Parse JSON → extract metrics
    └─ ResultTable.load() → DataTable.add_columns() + add_rows()
    ↓
  Metrics displayed in tabular format
