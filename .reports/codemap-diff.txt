================================================================================
ARCHITECTURE CHANGE NOTE
================================================================================

Before: Two API gateways
  LLM calls     → OpenRouter (https://openrouter.ai/api/v1)
  Embedding calls (OpenAI models) → OpenAI direct (https://api.openai.com/v1)

After: Single API gateway
  LLM calls     → OpenRouter
  Embedding calls → OpenRouter

The startswith("text-embedding") check in all 4 RAG framework pipelines routes
to the OpenAI-compatible embedding path. Model name stays "text-embedding-3-small"
(without "openai/" prefix) for framework compatibility.

The Phase 3 embedding benchmark models (openai_small.py, openai_large.py) use
the "openai/text-embedding-3-small" prefix as required by OpenRouter's embedding endpoint.
